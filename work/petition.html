원대협 서명운동 장려 페이지

사용자가 망설이지 않도록 “왜-어떻게-동의-서명” 흐름을 명확히 하고, 폼/동의 영역의 가독성과 실수 방지를 우선 설계했습니다.

Period 2024-10-03 - 2024-10-15
Role 디자인 제작 도움 · 프론트엔드 작업
Deliverables 랜딩 구조 · 폼 UI · 동의 영역 가독성 개선
Disclosure 캡처 기반 공개(비식별 처리)

Goal

  • 서명 참여까지의 단계(이해-동의-입력-제출)를 끊기지 않게
  • 개인정보 동의 영역을 “읽히는 형태”로 정리
  • CTA가 명확하고 반복되는 전환형 구조 유지

Key Decisions

신뢰 요소를 먼저 배치

참여 수치/참여 주체/안내 문구를 상단에서 바로 확인 가능하게 두어, 사용자가 “믿고 진행”할 수 있게 만들었습니다.

폼 UX 안정화

입력은 최소화하고, 필드 가이드/에러 방지/동의 체크 흐름을 끊기지 않게 구성했습니다.

동의 텍스트 가독성

긴 전문을 그대로 두면 읽히지 않기 때문에, 구획/여백/타이포를 조정해 “읽을 수 있는 덩어리”로 만들었습니다.

CTA 일관성

사용자에게 필요한 행동이 항상 1가지로 보이도록 버튼/문구/강조 색을 일관되게 유지했습니다.

Outcome

여기는 사실 기반으로만 작성: 예) 참여 완료율 변화(있다면), 문의 감소, 운영 대응 시간 단축, QA 이슈 감소 등.
