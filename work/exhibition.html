2023-2025 디자인국제전시회 전시장 UI

전시품/작가 정보가 많아도 “찾기-읽기-넘기기”가 가능한 탐색 구조로 정리하고, 운영 관점에서 반복 UI를 템플릿화했습니다.

Period 2023-2025
Role 디자인 제작 도움 · 프론트엔드 작업
Deliverables 전시장 UI(목록-상세) · 탐색 구조 · 운영 템플릿
Disclosure 캡처 기반 공개(비식별 처리)

Goal

  • 작품/작가가 많아도 사용자가 원하는 전시물을 빠르게 찾을 수 있게
  • 목록-상세 흐름이 자연스럽게 이어지도록
  • 매년 운영/업데이트를 고려해 반복 UI를 재사용 가능하게

Constraints

  • 콘텐츠 규모 증가(작가/작품 수, 전시 회차 누적)
  • 정보 단위가 다양함(프로필, 작품 이미지, 설명, 전시 맥락)
  • 운영 일정에 맞춘 업데이트 필요

Key Decisions

탐색 구조 우선

“어디를 보고 있는지”가 항상 유지되도록, 카테고리/내비게이션/목록 밀도 조절로 탐색 부담을 줄였습니다.

목록-상세 패턴 정리

상세 정보는 읽기 흐름을 해치지 않도록 카드/본문 구성을 분리하고, 이미지-텍스트 우선순위를 고정했습니다.

운영 템플릿화

반복되는 모듈(헤더/리스트/상세 섹션)을 재사용 가능하게 구성해, 회차 변경 시 수정 비용을 낮췄습니다.

프론트 구현 관점

반응형/QA/에셋 정리를 고려해 “깨지기 쉬운 구간”을 최소화하고, 유지보수 가능한 구조를 선택했습니다.

Outcome

정량 지표가 공개 불가한 경우, 운영 관점 성과(업데이트 시간 절감, QA 이슈 감소, 재사용 범위 확대 등)로 작성하는 것을 권장합니다. 여기는 네가 쓸 수 있는 사실만 넣어 주세요.
